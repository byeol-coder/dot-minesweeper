[닷 지뢰찾기 (Dot Minesweeper)] dot-games-host · 시각장애 사용자 우선 음성·촉각 설계  2026-07-21

■ 접근성 우선 UX (눈 없이 바로 이해·플레이)
  · 방향키/패닝으로 이동 → 매번 "3행 5열, 미공개" 처럼 위치+상태 음성. 격자 끝에선 "맨 위 줄이에요" 등 경계 안내.
  · 열기(엔터/F1) → "숫자 3, 주변 지뢰 3개" / "빈 칸, 여러 칸이 열렸어요" / "지뢰를 밟았어요" 처럼 결과를 명확히.
    빈 칸(0)은 인접 안전 칸이 자동으로 함께 열림(flood). 첫 열기는 항상 안전(3×3 안전지대).
  · 깃발(F/F2) → "깃발 표시, 남은 지뢰 7개" 처럼 남은 지뢰 수를 함께 안내.
  · 언제든: C 현재 칸 / R 현황(남은 지뢰·시간·연 칸) / B 판 전체 읽기(행별) / H 도움말 / N 새 게임.
  · 화면에도 좌표 라벨(행 1~5·열 1~10), 진행 HUD(남은 지뢰·시간·연 칸/안전 칸), 하단 조작 안내 상시 표시(저시력·보조자용).

■ 조작 요약
  화면/키보드: 방향키 이동 · Enter/Space 열기 · F 깃발 · C 현재 칸 · R 현황 · B 판 읽기 · H 도움말 · N 새 게임.
              (마우스: 클릭 열기 · 우클릭 깃발)
  DotPad: Panning 이동(행 우선 순차) · F1 열기 · F2 깃발 · F3 현재 칸 · F4 판 읽기 · PanningAll 도움말 · LPF1 새 게임.
  DotPad 촉각: 미공개=사각 테두리, 숫자=점 개수(1~8), 깃발=가운데 점, 지뢰=칸 채움, 커서=칸 안쪽 사각.
              남은 지뢰 수는 점자 숫자(상단 텍스트 라인).

■ 난이도: 쉬움(지뢰 4) · 보통(지뢰 6·기본) · 어려움(지뢰 8). 격자는 항상 6×5(30칸, 칸당 10×8핀)로 60×40 핀 면에 꽉 참.
         칸을 크게 잡아(닷 배틀십과 동급) 미공개·숫자·깃발·지뢰·커서 글리프를 손끝으로 확실히 구분.

■ 구조(외부 자산 표준): dotmine/ = index.html + dotpad-sdk/(DotPadSDK-3.0.0.js)
  · TTS: 호스트 루트의 /tts.js (TW_TTS) 사용, /api/tts 서버 음성 + 브라우저 음성 폴백.
  · SDK: dotpad-sdk/DotPadSDK-3.0.0.js 를 ES 모듈로 동적 import.
  · 임베드: ?embed=1 시 상단 설명 숨김, 부모창에 postMessage({source:'dotarcade', ...}) 로 ready/gameover 전달.
  · 인코딩/불변식: EA·encodeFrame(600hex) 그대로. 격자 6×5, 칸 10×8핀.

■ 배포: dotmine/ 폴더째 업로드(폴더명 반드시 'dotmine') → 재배포 → 하드 새로고침.
  확인: dot-games-host.vercel.app/dotmine/index.html?embed=1&preview=1
